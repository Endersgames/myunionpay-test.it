<analysis>**original_problem_statement:**
The user wants to create a Progressive Web App (PWA) for peer-to-peer payments that is not published on the Apple or Google Play stores but is self-installable from the browser.

**PRODUCT REQUIREMENTS:**
- **Core Functionality:** Peer-to-peer payments using a QR code system. Each QR code serves as both a referral link and a payment ID.
- **Wallet:** Each user has a virtual wallet with a balance denominated in UP points (not Euros).
- **Referral System:** When a new user registers via a referral link, both the referrer and the new user receive 1 UP in their wallets.
- **Unified QR Code:** A single QR code per user should handle two scenarios:
    1.  If scanned by an existing, logged-in user, it opens the payment interface to send funds to the QR code owner.
    2.  If scanned by a new user (or someone without the app), it directs them to the registration page with the referral code pre-filled. After registration, it should redirect them to the payment page.
- **PWA & Installation:** The app must be a PWA, prompting users to Add to Home Screen. This process should be seamless, especially on Chrome. For unsupported browsers (like the user's ZTE default browser), it should prompt the user to open the app in Chrome.
- **Marketplace:** An in-app marketplace for affiliated businesses/merchants, searchable by category.
- **Notifications:**
    - Merchants can send profiled and remunerated notifications to users.
    - The notifications should be real Push Notifications that make the phone ring or vibrate, even when the app is closed.
    - Merchants can target users based on interest tags or send broadcast notifications to all users.
- **User Profile & Gamification (New Request):**
    - A Task section where the first task is Upload your invoice and earn 5 UP.
    - A Feed section.
    - A progression bar that unlocks levels based on the user's UP balance:
        - 0-49 UP: (Base Level)
        - 50-149 UP: Energy Specialist
        - 150-779 UP: Trusted Energy
        - 780+ UP: Share Energy
- **Authentication:** User registration and login via email and phone number.
- **UI Request:** The Made with Emergent badge needs to be removed from the application.
